Working for Alderbrooke

A software engineering internship in summer 2018

I, alongside Max Campman, joined Alderbrooke for 8 weeks over summer after meeting their Chief Behavioural Scientist, Hani Nabeel. From that discussion we found out that Alderbrooke had an amazing set of scientific products that used quantitative psychology to assess the culture, talent and team dynamics of a company and improve them for the benefit of the company.

However, it was clear that Alderbrooke was struggling to implement the technology needed to keep up with their customer's ever increasing demand. So the aim of our internship was to completely overhaul the current production system of 'TalentScope', which was offline and used third party software, and re-write it as a cloud-based web application that was able to handle continuous processing and clients and analytics of their data.

What was the start point?

TalentScope is a tool used to assess a candidate's suitability for a job. Candidates are put through a psychological exam to measure them to quantitatively assess their personality type. The employer is also surveyed to produce a 'job profile' for the position. The analysis section of the system then assessed the candidate through the filter of the job profile and produced a final report of their suitability for the role.

A basic, 'offline' implementation of TalentScope was in operation. It surveyed the employer and candidates by using FormStack, but could not implement the adaptive 'Item Response Theory' section of the questionnaire which asked different questions depending on the previous responses, and was key to evaluating the accuracy of the later analysis. The results of every response then had to be downloaded by an analyst, processed using Excel, and reformatted into a report. This process would take 5-10 minutes per responder and was very unpopular with the analysts!!

What did we produce?

In 5 weeks, the two of us produced our web-app replacement which combined all features of TalentScope into one cloud-based platform:

  • Questionnaires now implemented Item Response Theory, so the results of the system were far more statistically significant.
  • Candidates and Job Profiles could now be associated to a company, so if a candidate could be assessed for an alternative role if they did not suit the one they initially applied for
  • The web-app has a whole admin section that allows analysts to quickly manage companies, jobs and candidates (including comply with GDPR); distribute links for questionnaires; upload the questionnaires in multiple languages; and most importantly generate reports immediately.
  • This admin site is secured against unauthorised access since the platform contains sensitive information.
  • Reports can be generated in one-click! Any candidate can be assessed for any job in the same company using the same test results. Our analytics system provided immediate processing and report generation - 5 minutes down to no time at all!

What we learnt from this project

When we started this project, we were strong programmers (especially in Java), great mathematicians, and had studied a number of relevant theoretical courses such as Database Design, Software and Security Engineering, Algorithms, and so on. But we had little web-development experience, not much team experience, and no industry experience at all! Suddenly, we had users to gather requirements from; stakeholders who were dependant on its success; expectations that it would work, be easy to use, be delivered on time. All of which with no other developers in the company to show us the ropes - we were learning it all by ourselves!

Our first challenge was in getting the requirements. Alderbrooke is small and everyone is very busy so we had to approach every meeting with a plan to get all the information we needed out of them. There was also a lot left to our 'good judgement' which obviously required us to understand the product and decide what is best. One example was our decision to add the ability to upload questions in other languages; CultureScope has already gone global and TalentScope could go that way soon and so even though we were told to save time and not add it, the feature was really appreciated when we did it anyway!

Once we had the what, it was time to work out the 'how'. A full list of what we used is below and be assured we taught ourselves how to use nearly everything on there in the 5 weeks! In fact, the list of technologies we considered using is even longer (e.g. Scala, MongoDB, Python for NumPy...), so we were Tech Analysts, as well as Product Owners, Architects, Developers, Testers and completed the deployment - the real Full Stack! For a few days we did nothing but Google and Google until we understood how we were going to build this thing!

Technologies we used

  • Java
  • Spring Boot
  • Spring Security
  • MySQL: we designed the database architecture, interaction with it and the app, and used an SQL engine to efficiently compute stages of the analysis (especially aggregation calculation). I take particular pride in how we designed our database to trade off normalisation for speed in some cases that were going to be read-heavy (e.g. the data used map the analysis calculations), and make other parts strongly normalised for high write-throughput.
  • HTML, CSS, Bootstrap and Thymeleaf
  • JavaScript
  • Maven and Git for project management
  • JUnit: we built a full regression testing library for our system... and reaped the rewards!
  • AWS: EC2, Elastic Beanstalk, RDS, Certificate Manager
  • SSL certification to provide https protocol for the site
